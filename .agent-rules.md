# SUMM-Powers Worker Agent Rules

## 项目背景

SUMM-Powers 是一个 skills 框架，fork 自 obra/superpowers。
核心产物是 Markdown skill 文件，供 Claude Code、Gemini CLI、Copilot CLI 使用。

## 编码规范

- Shell 脚本使用 `#!/usr/bin/env bash` 和 `set -euo pipefail`
- 所有文本文件使用 LF 换行（`.gitattributes` 已配置）
- Skill 文件路径：`skills/<name>/SKILL.md`，必须有 YAML frontmatter（`name`, `description`）
- 插件版本在 `.claude-plugin/plugin.json`，遵循 `<upstream-version>-summ.<fork-version>` 格式

## 工作流程

1. 先读相关文件，理解现有代码结构和上下文
2. 列出要修改的文件清单，确认改动范围
3. 实现改动，每一步确保不破坏现有功能
4. 编写或更新测试用例
5. 运行测试验证
6. 创建 PR，标题用 conventional commit 格式（feat:, fix:, refactor: 等）

## 测试要求

```bash
# 快速测试（必须通过）
./tests/claude-code/run-skill-tests.sh

# 单个测试
./tests/claude-code/run-skill-tests.sh --test test-subagent-driven-development.sh

# Brainstorm server 测试
cd tests/brainstorm-server && npm test
```

- 新 skill 必须有对应测试
- 修改已有 skill 要跑相关测试确保不破坏
- PR 提交前必须运行测试套件
- 测试使用 `claude -p`（headless 模式）验证 skill 加载和 agent 行为

## Skill 编写规范

遵循 `skills/writing-skills/SKILL.md` 的 TDD 流程：
1. 写压力测试场景（pressure test）
2. 验证基线失败
3. 编写 skill 内容
4. 验证 skill 通过测试
5. 重构优化

## Git 规范

- 从 main 分支创建 feature 分支，命名格式：`feat/<描述>` 或 `fix/<描述>`
- Commit message 使用 conventional commits
- PR 描述中关联 issue（自动关闭）
- 一个 PR 只做一件事

## 禁止事项

- 不修改 `.gitattributes`
- 不直接修改 `.claude-plugin/plugin.json` 版本号
- 不删除已有测试用例（可以修改内容）
- 不修改 `hooks/session-start` 除非明确要求
- 不在 skill 文件中使用 emoji
- 设计规范放 `docs/superpowers/specs/`，实现计划放 `docs/superpowers/plans/`

## 目录结构速查

```
skills/<name>/SKILL.md       — Skill 定义
commands/                     — Slash 命令（调用 skill 的薄包装）
agents/                       — Agent 定义（subagent 派发用）
hooks/session-start           — SessionStart hook
.claude-plugin/plugin.json    — 插件清单
tests/claude-code/            — 测试套件
tests/brainstorm-server/      — Brainstorm server 测试
docs/superpowers/specs/       — 设计规范
docs/superpowers/plans/       — 实现计划
external-skills/              — 已废弃，外部 skill 现在加载到 skills/ext-<id>/
```
