# 编写 ARCHITECTURE.md

## 概述

ARCHITECTURE.md 弥合了偶尔贡献者和核心开发者之间的认知差距。最大的差异不在于写代码——而在于找到该在哪里改代码。这份文档回答两个问题："做 X 的东西在哪？"和"我看的这个文件是干什么的？"

**核心原则：** 代码地图是一个国家的地图，不是其各州地图的合集。

## 适用场景

- 为新项目或现有项目创建 ARCHITECTURE.md
- 修订现有的 ARCHITECTURE.md（每年 revisiting 几次，不是每次提交都改）
- 跳过：项目不到 ~1k 行代码、CLAUDE.md 已经充分覆盖架构

## 结构

每份 ARCHITECTURE.md 应按顺序包含以下章节：

### 1. 全局概览

用 5-10 行描述系统要解决的问题和高层的数据流。读者读完这一节就能理解系统的目的和数据如何流转。

### 2. 代码地图

对每个顶层目录或主要模块：
- 它包含什么、扮演什么角色（2-3 行）
- 按名称列出关键文件或类型（但参见下方规则）
- 该模块特有的架构不变量

### 3. 层边界

显式描述层与层之间的边界。对每个边界，说明每一边对另一边不知道什么。这是对新贡献者最有价值的章节——边界约束了所有可能的实现，但靠读代码很难发现边界。

即使是单体项目也有概念上的层（入口 → 业务逻辑 → 数据）。描述那些。如果项目确实没有分层，解释为什么这是有意为之。

### 4. 核心不变量

跨模块的系统级不变量。特别要指出以"缺席"形式表达的不变量（"X 从不依赖 Y"、"A 层的模块不知道 B 层的存在"）。

### 5. 横切关注点

测试策略、错误处理、可观测性、构建系统——无处不在又无处所在的东西。

## 规则

### 应该做

- **命名重要的文件、模块和类型。** 使用精确名称以便读者用 symbol search 查找。
- **鼓励 symbol search。** 在文档顶部附近加一句提示："用 symbol search 查找文中提到的实体。"
- **标出架构不变量。** 特别是代码中"缺席"的东西——"X 层不依赖 Y 层"在读 X 层代码时是不可见的。
- **显式描述边界。** 说明每层不知道其邻居的什么。
- **反思物理布局。** 如果逻辑上相关的东西在物理上不相邻，指出这种张力并解释原因。
- **保持简短。** 目标 80-120 行。每个反复贡献者都要读它。
- **只包含稳定的事实。** 避免版本号、具体 commit hash 或每次发布都会变的细节。

### 不应该做

- **不要链接文件。** 链接会过期。使用文件名，让读者自己搜索。
- **不要解释每个模块如何工作。** 那是内联文档或独立文档的事。代码地图说的是东西"是什么"和"在哪里"，不是"怎么运作"。
- **不要枚举内部文件。** "包含辅助 prompt"就够了——不要列出每个 prompt 文件名。
- **不要包含特定版本的值。** "版本遵循 X 模式"是稳定的；"当前版本是 5.0.7"不是。
- **不要写第二个 README。** 这是给贡献者的，不是给用户的。聚焦于代码导航，不是功能描述。

## 常见错误

| 错误 | 为什么有问题 | 修正方式 |
|------|-------------|---------|
| 代码地图列出了模块中的每个文件 | 地图变成了合集——难维护，容易过时 | 命名 2-3 个关键项，概括其余 |
| 没有层边界章节 | 新人最需要的信息缺失 | 加上显式的"X 不知道 Y"声明 |
| 包含版本号或 hash | 每次发布都会变，文档变错 | 描述模式或约定 |
| 解释实现细节 | 属于内联文档，撑大文档 | 只描述"是什么"和"在哪"，不描述"怎么运作" |
| 面向用户而非贡献者 | 受众错误——这是贡献者的地图 | 聚焦于代码导航，不是功能描述 |
| "我的项目没有层" | 即使单体也有入口 → 逻辑 → 数据 | 描述概念层；如果真的扁平，说明为什么 |
| "多列几个文件帮助读者找东西" | 撑大文档，很快过时 | 命名 2-3 个关键项；读者用 symbol search 找其余 |

## 质量检查清单

完成前，验证：

- [ ] 全局概览：新人能在 30 秒内理解系统？
- [ ] 代码地图：对每个主要模块能回答"X 在哪？"？
- [ ] 代码地图：对每个顶层目录能回答"这个目录是干什么的？"？
- [ ] 不变量：缺席型不变量（"X 从不依赖 Y"）是否显式声明？
- [ ] 边界：每个边界是否说明了每边不知道什么？
- [ ] 稳定性：没有版本号、hash 或每次发布都变的值？
- [ ] 长度：120 行以内？
- [ ] 没有链接，只用名称供 symbol search？
- [ ] 物理/逻辑布局的张力是否在存在时指出？
